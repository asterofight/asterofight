Astero Fight - Join us!

Free MMO space shooter by many of us

Come and join the team to make the game better! We need you if you find any of the following open topics interesting and can work on it.

You can join the team by applying to the university project lab Asterofight at Budapest University of Technology and Economics, Department of Automation and Applied Informatics; or if not a student then by sending an email to asterofight@outlook.com. Mention which topic you like the most.

Particle System

Programming - JavaScript (TypeScript)

The game server sends effect information to the client like explosions, beams, special fields, etc. These are either animations at a given point or if two points are connected then should be particle system.

The task is to design and implement a particle system with the following features.

  • No popping: The particle system should avoid sudden changes to not bother the player. It should be able to play fluid, continues animation, motion even when effects appear and disappear at any time, or start from 50% if they just got into the view.
  • Scalable: The particle system should have a global parameter to specify particle density to support low power clients.
  • Support moving and stationary emitters
  • Optional: Occlusion detection and repression (stronger particle can hide weaker underneath and as such can be omitted from draw calls)
  • Optional: Occlusion detection with animation subsystem

Network Lag Compensation

Programming - JavaScript (TypeScript)

The game server sends one packet every 200 ms. The network latency and lag makes this between 0 and 1000 as observed by the client. The task is to compensate this with the movement of objects so they do not jump around. At the moment a simple PID controller does the job quite well but it is not flawless.

The task is to design and implement a controller to move objects on the screen with the following features.

  • Objects should not teleport or lag behind;
  • Objects should be as close (distance) to their server sent position as possible;
  • Objects should be as close (in time) to real time as possible - should be as responsive as possible;
  • Implement a measurement system to evaluate diffrent algorithms and parameters;

User Interface, HUD

Graphics and Design

The game has a very basic UI which badly needs a better design.

The task is to design and draw a new User Interface for the game with the following features.

  • Support PCs and big monitors (full featured);
  • Support tablets and smaller laptops (might be full featured but can omit certain UI elements);
  • Support phones in landscape mode (limited features);

Game Objects

Graphics and Design

The game has barely any graphics for objects. We have approx 20 spaceships, asteroids, missiles.

The task is to design and draw all/some game objects with the following features.

  • 3D looking, 2D rendered images;
  • On a 100 dpi monitor the small spaceships are around 30 pixels
  • Support for high DPI screens;

Bots (AI)

Programming - C#

The game uses bots to substitute players and give a different game play. These are server side algorithms that control spaceships.

The task is to design and implement artificial intelligent (AI) controlled spaceships with the following features.

  • Limited processor time: max 100 µs (0.1 ms) run time per Update;
  • Reasonable survivability;
  • Should be able to move on the map to hot spots (like middle) and avoid asteroids;

Agents (AI)

Programming - JavaScript (TypeScript)

Act as a player and use a browser or console to connect to server and drive an agent that has one goal to be better than players. Compete against other agents and players and win on the monthly competition!

The task is to design and implement artificial intelligent (AI) controlled spaceship with the following features.

  • Processor time is only limited by client computer;
  • Network lag applies;
  • Can act every 200 ms;
  • May control more than one spaceship, they are allowed to communicate;
